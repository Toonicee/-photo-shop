  1. alert синтаксис
    1. alert(сообщение)
  2. prompt синтаксис
    1. result = prompt(title, default);
  3. confirm синтаксис
    1. result = confirm(question);

alert выводит сообщение

prompt выводит сообщение и ждет, пока пользователь введет текст, а затем возвращает введенное значение или null, если ввод отменен (CANCEL/esc)

confirm выводит сообщение и ждет, пока пользователь нажмет ОК или CANCEL и возвращает true/false

Условные операторы: if, "?"

Оператор if (если) получает условие. Он вычисляет его, и если результат - true, то выполняет команду.

Неверное условие, else ("иначе")выполняется, если условие не верно

Рекомендуется использовать фигурные скобки всегда, даже когда команда одна. Это улучшает читаемость кода.

Логические операторы

|| (ИЛИ)

Оператор ИЛИ выглядит как двойной символ вертикальной черты:

result = a || b;

Логическое ИЛИ в классическом программировании работает следующим образом: "если хотя бы один из аргументов true, то возвращает true, иначе – false". В JavaScript, как мы увидим далее, это не совсем так, но для начала рассмотрим только логические значения.

Короткий цикл вычислений

JavaScript вычисляет несколько ИЛИ слева направа. При этом, чтобы экономить ресурсы, используется так называемый корготкий цикл вычисления

Допустим, вычисляются несколько ИЛИ подряд: a || b || c || Если первый ргумент - true, то результат заведомо будет true (хотя бы одно из значений - true), и остальные значения игнорируются.

Это особенно заметно, когда выражени, переданное в качестве второго аргумента, имеет сторонний эффект- например, присваевание переменную.

Итак, оператор || вычисляет операнды слева направо до первого "истинного" и возвращает его, а если все ложные - то последнее знчение.

Иначе можно сказать, что || запинается на пвавде

&& (И)

Оператор и пишется как два оперсанда &&

result = a && b;

В классическом программировании И возвращает true, если оба аргумента истинны, а иначе – false

К И применим тот же принцип "короткого цикла вычислений", но немного по-другому, чем к ИЛИ.

Если левый аргумент - false, оператор И возвращает его и заканчивает вычисления. Иначе - вычисляет и возвращает правый аргумент.

Итак, оператор && вычисляет операнды слева направо до первого «ложного» и возвращает его, а если все истинные – то последнее значение.

Иначе можно сказать, что "&& запинается на лжи".

Приоритет оператора И && больше, чем ИЛИ ||, так что он выполняется раньше.

! НЕ

Операнор НЕ -самый простой. Он получает один аргумент

var result = !value;

    Действия !
  1. Сначало приводит аргумент к логическому типу true/false
  2. Затем возвращает противоположное значениею.

Циклы while, for

Циклы while

циклцршду имеет вид:

while (условие){
// код, тело цикла
}

Пока условие верно - выполнряется код из тела цикла.

Например, цикл ниже выводит i пока i < 3:

var i = 0;
while (i < 3) {
alert(i);
i++;

Повторение цикла по-научному называется «итерация».

Цикл for

Чаще всего применяют цикл for.

for (начало; условие; шаг){
//...тело цикла...
}

Пример цикла

var i;
for (i = 0; i < 3; i++) {
alert( i );
}

Цикл выполняется так

  1. Начало: i=0 выполняется один-единственный раз, при заходе в цикл.
  2. Условие: i<3 проверяется перед каждо интерацией и при входе в цикл, если оно нарушено, то происходит выход.
  3. Тело: alert(i).
  4. Шаг i++ выполняется после тела на каждый итерации, но перед проверкой условия.
  5. Идти на шаг 2.

В цикле также можно определить переменную

for (var i = 0; i < 3; i++) {
alert(i); //0, 1, 2
}

Любая часть for может быть пропущена.

Прерывание цикла: break

Выйти из цикла можно не только при проверке условия но и, вообще, в любой момент.Это возможность обеспечивает директива break

Следующая итераия: continue

Директива continue прекращает выполнени текущей интерации цикла

Она – в некотором роде «младшая сестра» директивы break: прерывает не весь цикл, а только текущее выполнение его тела, как будто оно закончилось.

Итого

JavaScript поддерживает три вида циклов:

  • while – проверка условия перед каждым выполнением.
  • o..while – проверка условия после каждого выполнения.
  • for – проверка условия перед каждым выполнением, а также дополнительные настройки.
